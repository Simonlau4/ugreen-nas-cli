# UGREEN NAS CLI 后续优化清单

这个工具的目标不是做成正式开源项目，而是让公司内部同事和他们自己的 AI Agent 稳定访问、稳定使用 NAS。后续优化优先解决真实使用里的连接、配置、权限和误操作问题。

## 优先级 1：先把连接稳定下来

- [ ] 固定推荐连接方式：优先 Tailscale / 公司 VPN，其次同一内网，暂不建议公网 DDNS 直连。
  - 验收：README 里有清楚的“同事怎么连上 NAS”的说明。
- [ ] 加强 `doctor` 检查。
  - 验收：能明确区分证书问题、账号密码错误、网络不通、WebDAV 未启用、目录不存在、权限不足。
- [ ] 增加 `doctor --path "/共享目录"`。
  - 验收：不仅检查 WebDAV 根路径，还能检查同事是否真的能访问指定目录。
- [ ] 对自签名证书给出更清楚的提示。
  - 验收：用户看到错误时知道是临时用 `--insecure`，还是应该找管理员配证书。
- [ ] 记录一份内部排错表。
  - 验收：常见错误能按提示处理，例如 `CERTIFICATE_VERIFY_FAILED`、`401`、`403`、`404`、连接超时。

## 优先级 2：让同事安装配置更顺手

- [ ] 做一个“同事版”最短安装说明。
  - 验收：只保留 clone、install、setup、doctor 四步。
- [ ] 优化 `scripts/setup-profile-macos-keychain.sh` 的交互提示。
  - 验收：同事不用理解 WebDAV 细节，也能按提示输入地址、用户名、共享目录和密码。
- [ ] 安装脚本检测 `~/.local/bin` 是否在 PATH，并给出可复制命令。
  - 验收：安装后如果命令不可用，提示能直接解决。
- [ ] 增加“更新工具”的说明。
  - 验收：同事知道以后怎么 `git pull` 后重新运行安装脚本。
- [ ] 准备一段可直接发群里的安装消息。
  - 验收：复制给同事即可，不包含任何你的账号、密码或内网地址以外的敏感信息。

## 优先级 3：降低 AI Agent 误操作风险

- [ ] 增加 `capabilities --json`。
  - 验收：AI Agent 能先读取允许的命令、危险命令和示例。
- [ ] 增加全局 `--dry-run` 或至少给 `put/mv/cp/rm/mkdir` 增加 dry-run。
  - 验收：AI Agent 可以先预览将要操作的路径。
- [ ] 删除目录时增加二次门槛。
  - 验收：删除目录需要额外参数，例如 `--recursive --yes`。
- [ ] 对大文件上传/下载给提示。
  - 验收：超过设定大小时提醒用户确认，避免 AI Agent 无意下载大视频。
- [ ] 强化 `AI_AGENT_USAGE.md`。
  - 验收：里面有一段可直接复制给 Agent 的短 prompt，强调先 `ls/stat`，再写入，不主动删除。

## 优先级 4：真实 NAS 使用体验

- [ ] 增加“只读冒烟测试”命令。
  - 验收：可以快速检查 `doctor + ls + search`，不创建文件。
- [ ] 增加“读写冒烟测试”命令。
  - 验收：创建临时目录、上传、读取、复制、移动、删除，并自动清理；清理失败要报告路径。
- [ ] 优化回收站行为说明。
  - 验收：用户知道删除后 NAS 可能进 `#recycle`，以及是否需要手动清理。
- [ ] 增加本地审计查看命令。
  - 验收：能查看最近执行过哪些写入、移动、删除操作。
- [ ] 增加更稳定的搜索策略。
  - 验收：搜索大目录时能限制深度、数量，并显示是否达到上限。

## 优先级 5：小团队维护

- [ ] 保留 public repo，但 README 面向“内部小团队使用”，不要过度包装成通用开源项目。
  - 验收：文案清楚、实用，不强调发布、license、社区贡献。
- [ ] 每次推送前跑本地测试和敏感信息扫描。
  - 验收：不把账号、密码、真实内网细节误推到 public repo。
- [ ] 记录当前推荐配置模板。
  - 验收：有一份不含密码的示例，说明 `base_url`、`allowed_root`、`--insecure` 怎么填。
- [ ] 保持命令少而稳。
  - 验收：除非真实需要，不新增 SFTP/SMB 等第二协议。

## 当前最应该做的 5 件事

1. 优化 `doctor`，把错误提示做清楚。
2. 增加 `doctor --path`，确认指定共享目录可访问。
3. 加一个读写冒烟测试命令，自动创建和清理临时文件。
4. 简化 README，改成公司内部同事使用导向。
5. 给 AI Agent 使用文档加一段可复制 prompt。
